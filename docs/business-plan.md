## 1. 项目背景

### 1.1 为什么要做这个产品

本项目要创造一个面向儿童的中文互动阅读 Web App。

核心目标不是把纸质书简单搬到屏幕上，而是通过数字化方式降低孩子的阅读门槛，让孩子在较短时间内完成一次完整的阅读训练。

目标问题包括：

- 孩子对纯文字阅读兴趣不足
- 认字和阅读理解之间缺少连续训练
- 家长希望孩子每天阅读，但陪读成本较高
- 孩子读完故事后缺少即时反馈
- 阅读行为缺少持续激励

因此，第一阶段产品要验证的是：

> 孩子是否能通过短故事、识字练习、阅读理解题和星星激励，完成一次 5–10 分钟的中文阅读训练。

### 1.2 当前参考方向

当前参考方向是“叫叫”类儿童阅读/成长数字内容产品。

参考的是产品方向和能力结构，包括：

- 分龄或分级内容
- 故事化阅读
- 图文结合
- 阅读后的练习反馈
- 识字与阅读理解结合
- 星星、徽章、成长等级等激励机制
- 每日短时学习任务

但本项目不复制叫叫的具体内容、IP、课程、图片、音频、题库或页面表达。

### 1.3 我们不做什么

第一阶段不做完整教育平台，也不做叫叫复制版。

第一阶段不做：

- 不做叫叫内容复制
- 不做商业课程包
- 不做真人老师服务
- 不做复杂家长端
- 不做支付系统
- 不做原生 iOS / Android App
- 不做大量动画
- 不做语音识别评分
- 不做 AI 作文批改
- 不做用户社区
- 不做 UGC 内容发布
- 不做后台 CMS

第一阶段只验证一个最小闭环：

> 读一篇原创短故事 → 完成识字与阅读理解题 → 获得星星 → 记录完成进度。

## 2. 目标用户

### 2.1 核心用户

核心使用者：

```text
6–8 岁儿童
```

这个年龄段已经具备一定识字基础，适合做中文自主阅读、识字巩固和基础阅读理解训练。

用户特点：

| 年龄 | 阅读特点 | 产品侧重点 |
|---|---|---|
| 6 岁 | 识字量增长中，仍需要图文辅助 | 短句阅读、关键词识别、简单理解题 |
| 7 岁 | 可以阅读简单故事 | 分页阅读、基础因果理解、词语理解 |
| 8 岁 | 能读完整短故事 | 情节理解、人物行为判断、简单推理 |

第一版重点聚焦：

```text
6–8 岁儿童的中文阅读与认字训练。
```

### 2.2 付费/决策用户

决策者是家长，不是孩子。

目标家长画像：

- 希望孩子养成中文阅读习惯
- 希望孩子提升识字量和阅读理解能力
- 希望每天学习时间可控，不要太重
- 希望产品不是纯娱乐游戏
- 希望内容安全、简单、无广告
- 希望第一阶段无需复杂账号和隐私授权

第一阶段不做商业化，但业务设计要保留未来扩展空间。

### 2.3 使用场景

核心场景：

| 场景 | 描述 |
|---|---|
| 放学后阅读 | 孩子完成一篇短故事阅读和题目 |
| 晚饭后学习 | 家长安排孩子做 5–10 分钟中文阅读训练 |
| 周末自主阅读 | 孩子自己选择故事并完成闯关 |
| 家长陪伴阅读 | 家长陪孩子看故事，并查看答题反馈 |

第一阶段主场景：

```text
孩子每天完成一篇 5–10 分钟的中文互动阅读任务。
```

## 3. 用户问题

### 3.1 孩子的问题

孩子的问题：

- 不爱看长文字
- 认识一些字，但不能稳定理解完整句子
- 阅读时容易跳字、漏字
- 读完故事后不知道重点
- 缺少即时反馈
- 缺少持续阅读动力

产品要解决的是：

```text
让阅读变短、变清晰、变有反馈，并把认字和理解结合起来。
```

### 3.2 家长的问题

家长的问题：

- 想培养孩子阅读习惯，但每天陪读成本高
- 不知道孩子是否真的读懂
- 不知道内容是否适合当前年龄
- 希望孩子认字，但不希望产品变成枯燥识字卡
- 担心普通 App 娱乐性太强
- 担心儿童数据和隐私问题

产品要解决的是：

```text
让家长可以放心地把一个短阅读任务交给产品完成。
```

### 3.3 教育内容的问题

内容问题：

- 只有故事，没有识字巩固
- 只有识字，没有阅读语境
- 只有题目，孩子缺少兴趣
- 内容太难，孩子读不下去
- 内容太简单，孩子没有成长

产品策略：

```text
故事是入口，认字是基础，理解是目标，激励是驱动。
```

## 4. 业务定位

### 4.1 产品一句话定位

```text
一个面向 6–8 岁儿童的中文互动阅读 Web App，通过原创短故事、识字练习、阅读理解题和星星激励，帮助孩子每天完成 5–10 分钟阅读训练。
```

更短版本：

```text
儿童中文互动阅读与认字训练软件。
```

### 4.2 和叫叫类产品的关系

关系定义：

```text
参考叫叫类产品的业务方向、功能结构和用户体验，但不复制其内容、IP、课程、题库、素材和页面表达。
```

可以参考：

- 分级阅读思路
- 故事化学习
- 互动反馈
- 激励机制
- 每日任务
- 图文结合的阅读体验

不能复制：

- 叫叫品牌
- 叫叫 IP
- 叫叫课程
- 叫叫故事文案
- 叫叫图片/音频/视频
- 叫叫题库
- 叫叫商业包装
- 叫叫页面布局和视觉表达

### 4.3 差异化方向

第一阶段不追求商业差异化，先追求可控、轻量、可持续开发。

差异化暂定为：

| 方向 | 说明 |
|---|---|
| 轻量 | Web App 原型优先，不做重 App |
| 原创内容 | 使用原创短故事，降低版权风险 |
| 阅读+认字 | 不是单纯讲故事，也不是孤立识字 |
| 任务化 | 每天一篇短故事，降低学习压力 |
| 低干扰 | 不做广告、不做社区、不做过强游戏化 |
| 可扩展 | 后续可扩展 AI 故事、音频、家长端 |

## 5. 核心业务闭环

### 5.1 阅读闭环

```text
选择故事
→ 阅读分页内容
→ 识别重点字词
→ 完成整篇故事
```

目标：

- 孩子能读完
- 每页内容不要太长
- 重点字词有明确呈现
- 阅读过程尽量简单

### 5.2 学习闭环

```text
阅读故事
→ 回答识字题
→ 回答阅读理解题
→ 获得对错反馈
→ 完成学习记录
```

第一阶段题型：

- 单选题
- 判断题
- 识字选择题

每篇故事 3 道题：

```text
1 道识字题 + 2 道阅读理解题
```

暂不做：

- 主观题
- 作文题
- 语音答题
- AI 批改
- 复杂排序题

### 5.3 激励闭环

```text
完成阅读
→ 获得星星
→ 点亮故事
→ 累计完成记录
→ 回到首页继续下一篇
```

第一阶段激励只做轻量版：

- 星星
- 完成状态
- 简单连续阅读天数
- 简单徽章

不做复杂宠物养成、商城、金币消费系统。

## 6. 第一阶段业务范围

### 6.1 第一阶段要做

第一阶段做一个可运行 MVP。

范围：

| 模块 | 是否做 | 说明 |
|---|---|---|
| 首页 | 做 | 展示今日任务、进度、星星 |
| 故事列表 | 做 | 展示 5 篇原创短故事 |
| 阅读页 | 做 | 图文分页阅读，展示重点字词 |
| 答题页 | 做 | 每篇 3 题：识字题 + 阅读理解题 |
| 完成页 | 做 | 展示奖励和完成反馈 |
| 本地进度 | 做 | 保存完成状态和星星 |
| 内容数据 | 做 | 内置原创故事和题目 |
| 账号登录 | 不做 | 第一阶段本地存储即可 |
| 家长端 | 不做 | 后续再做 |
| 支付 | 不做 | 后续再做 |
| 后台 CMS | 不做 | 第一阶段内容写死在本地数据 |
| 后端服务 | 不做 | 第一阶段不需要服务器 |

### 6.2 第一阶段不做

明确不做：

```text
1. 不做完整课程体系
2. 不做复杂动画
3. 不做原生 App
4. 不做服务器后端
5. 不做支付订阅
6. 不做家长报告
7. 不做老师服务
8. 不做 AI 生成内容上线
9. 不做用户上传内容
10. 不做公开发布给儿童大规模使用
11. 不复制叫叫内容、IP、素材、题库或课程
```

### 6.3 第一阶段成功标准

第一阶段成功标准不是收入，而是原型可用。

验收标准：

| 类型 | 标准 |
|---|---|
| 主流程 | 首页 → 故事列表 → 阅读 → 答题 → 完成，可以跑通 |
| 内容 | 5 篇原创短故事 |
| 题目 | 每篇 3 道题，包含识字题和阅读理解题 |
| 激励 | 完成后可以获得星星 |
| 进度 | 刷新页面后仍能看到完成记录 |
| 体验 | 一个孩子可在 5–10 分钟内完成一篇 |
| 技术 | 本地可运行，可部署到网页 |

## 7. 内容策略

### 7.1 内容来源

第一阶段使用：

```text
原创短故事
```

不使用：

- 叫叫内容
- 商业绘本内容
- 未确认版权的图片
- 未授权音频
- 未授权 IP 角色
- 受版权保护的故事改写

第一阶段候选故事方向：

| 故事名 | 主题 | 识字方向 |
|---|---|---|
| 小兔子的红雨伞 | 分享与帮助 | 雨、伞、红、朋友 |
| 会发光的石头 | 好奇心 | 光、石、找、亮 |
| 小熊不想刷牙 | 生活习惯 | 牙、刷、水、干净 |
| 云朵邮递员 | 想象力 | 云、信、送、天空 |
| 森林里的慢慢车 | 耐心与合作 | 车、慢、等、一起 |

### 7.2 内容分级

第一阶段先做 3 个等级，不做复杂课程体系。

| 等级 | 目标年龄 | 内容特点 |
|---|---|---|
| L1 | 6 岁左右 | 短句、常见字、强图片辅助 |
| L2 | 7 岁左右 | 简单情节、基础因果理解、词语识别 |
| L3 | 8 岁左右 | 完整短故事、人物动机、简单推理 |

第一版可以优先实现：

```text
L1 + L2
```

L3 作为扩展方向。

### 7.3 内容版权边界

内容版权原则：

- 文案原创
- 题目原创
- 角色原创
- 图片使用自制、AI 生成且注意授权边界，或使用明确可商用素材
- 不使用叫叫素材
- 不使用知名儿童 IP
- 不直接改写仍受版权保护的绘本
- 公版故事如使用，也必须重新改写并注明来源策略

第一阶段建议全部原创，最安全。

## 8. 商业化策略

### 8.1 第一阶段是否商业化

第一阶段不商业化。

原因：

- 产品形态未验证
- 内容体系不完整
- 没有合规体系
- 没有支付和账号
- 不适合直接面向儿童正式运营

第一阶段定位：

```text
可演示、可体验、可迭代的 MVP 原型。
```

### 8.2 后续可能的商业模式

后续可选：

| 模式 | 说明 |
|---|---|
| 内容订阅 | 每月解锁更多故事 |
| 分级课程包 | 按 L1/L2/L3 售卖 |
| 家长报告 | 提供阅读能力和识字进度分析 |
| AI 故事生成 | 家长定制主题故事 |
| 机构合作 | 给幼儿园、低龄语文机构使用 |

以上都不是第一阶段目标。

## 9. 风险与约束

### 9.1 内容风险

风险：

- 内容不适龄
- 故事情节价值观不合适
- 题目设计过难或过简单
- 未授权素材侵权
- 误复制竞品内容

应对：

- 第一阶段只用原创短故事
- 每篇故事控制主题
- 不涉及暴力、恐怖、歧视、成人内容
- 图片素材来源可控
- 不复制叫叫内容、IP、课程、题库、图片、音频或页面表达

### 9.2 儿童产品合规风险

6–8 岁用户属于儿童群体。第一阶段必须降低数据和隐私复杂度。

第一阶段原则：

```text
不做真实儿童账号
不收集姓名、手机号、照片、语音
不上传儿童数据到服务器
只使用浏览器本地存储保存测试进度
```

如果未来正式运营，并涉及儿童个人信息，需要单独补充儿童隐私保护、监护人同意、数据安全和内容安全方案。

### 9.3 技术与成本约束

第一阶段限制：

- 只做 Web App
- 不做后端
- 不接数据库
- 不接支付
- 不接真实音频服务
- 不做复杂动画引擎
- 不做真实账号体系

这样可以确保项目先跑起来。
