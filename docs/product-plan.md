> 当前版本：v0.2  
> 当前状态：详细内容已确认  
> 写入规则：用户确认后，才允许写入 Git

## 1. 产品定位

### 1.1 产品一句话说明

```text
一个面向 6–8 岁儿童的中文互动阅读 Web App，通过原创短故事、识字练习、阅读理解题和星星激励，帮助孩子每天完成 5–10 分钟阅读训练。
```

更短版本：

```text
儿童中文互动阅读与认字训练软件。
```

### 1.2 产品目标

第一版产品目标不是做完整教育平台，而是验证一个最小阅读学习闭环：

```text
选择故事 → 阅读分页内容 → 完成识字题和阅读理解题 → 获得星星 → 保存完成进度
```

核心目标：

- 让孩子愿意打开并完成一篇短故事。
- 让孩子在故事语境中复习重点字词。
- 让孩子通过简单题目确认是否读懂。
- 让家长看到孩子是否完成阅读任务。
- 让产品在无账号、无后端的情况下先跑通 MVP。

### 1.3 第一版产品边界

第一版只做可运行 MVP，不做正式商业化运营版本。

第一版做：

- 首页
- 故事列表页
- 故事详情页
- 阅读页
- 答题页
- 完成页
- 本地学习进度
- 星星奖励
- 5 篇原创短故事
- 每篇 3 道题

第一版不做：

- 账号登录
- 支付订阅
- 后端服务
- 数据库
- 家长报告详情页
- 老师端
- 后台 CMS
- 语音识别评分
- AI 自动生成内容上线
- 用户上传内容
- 复杂动画和宠物养成
- 复制竞品内容、IP、课程、题库、图片、音频或页面表达

## 2. 目标用户与使用场景

### 2.1 目标用户

核心儿童用户：

```text
6–8 岁儿童
```

用户特点：

| 年龄 | 阅读特点 | 产品设计重点 |
|---|---|---|
| 6 岁 | 识字量增长中，需要图文辅助 | 短句、重点字词、简单选择题 |
| 7 岁 | 可以阅读简单故事 | 分页阅读、基础理解、词语识别 |
| 8 岁 | 能读完整短故事 | 情节理解、人物行为判断、简单推理 |

决策用户：

```text
家长
```

家长关注点：

- 内容是否适合孩子年龄。
- 孩子是否真的完成阅读。
- 学习时间是否可控。
- 产品是否安全、简单、无广告。
- 第一版是否不需要复杂账号和隐私授权。

### 2.2 核心使用场景

| 场景 | 用户行为 | 产品支持 |
|---|---|---|
| 放学后阅读 | 孩子完成一篇短故事 | 今日任务、故事入口、完成奖励 |
| 晚饭后学习 | 家长安排孩子做 5–10 分钟训练 | 简短流程、进度记录、星星反馈 |
| 周末自主阅读 | 孩子自己选择故事闯关 | 故事列表、完成状态、未完成提示 |
| 家长陪伴阅读 | 家长陪孩子读并看结果 | 答题反馈、完成页、本地记录 |

第一版主场景：

```text
孩子每天完成一篇 5–10 分钟中文互动阅读任务。
```

### 2.3 用户使用频率

第一版设计为轻量日常使用：

| 频率 | 说明 |
|---|---|
| 每天 1 次 | 推荐完成 1 篇故事 |
| 每次 5–10 分钟 | 包含阅读和答题 |
| 每篇 3 道题 | 1 道识字题 + 2 道阅读理解题 |
| 连续完成 | 可累计星星和连续阅读天数 |

## 3. MVP 核心范围

### 3.1 第一版必须做

| 模块 | 是否必须 | 说明 |
|---|---|---|
| 首页 | 必须 | 展示今日任务、总星星、连续天数、继续阅读入口 |
| 故事列表页 | 必须 | 展示 5 篇原创故事、等级、完成状态 |
| 故事详情页 | 必须 | 展示故事简介、重点字词、预计时间、开始按钮 |
| 阅读页 | 必须 | 分页阅读故事，展示重点字词 |
| 答题页 | 必须 | 每篇 3 题，支持选择和即时反馈 |
| 完成页 | 必须 | 展示星星奖励、答题结果、鼓励语 |
| 本地进度 | 必须 | 使用浏览器本地存储保存完成记录 |
| 内容数据 | 必须 | 内置 5 篇原创故事和题目数据 |

### 3.2 第一版不做

| 模块 | 第一版是否做 | 原因 |
|---|---|---|
| 账号登录 | 不做 | 降低隐私和开发复杂度 |
| 后端服务 | 不做 | MVP 本地即可跑通 |
| 数据库 | 不做 | 内容和进度先保存在前端 |
| 支付订阅 | 不做 | 第一版不商业化 |
| 家长报告详情页 | 不做 | 后续扩展 |
| 语音播放 | 可预留，不强制 | 第一版先不接真实音频 |
| 语音识别评分 | 不做 | 技术复杂度高 |
| AI 内容生成 | 不做 | 涉及内容安全和审核 |
| 复杂游戏系统 | 不做 | 防止偏离阅读目标 |

### 3.3 MVP 验收标准

| 类型 | 验收标准 |
|---|---|
| 主流程 | 首页 → 故事列表 → 故事详情 → 阅读 → 答题 → 完成，可以完整跑通 |
| 内容 | 至少 5 篇原创短故事 |
| 题目 | 每篇 3 道题，包含 1 道识字题和 2 道阅读理解题 |
| 阅读 | 支持分页阅读，能前进和返回 |
| 反馈 | 答题后能看到正确/错误反馈 |
| 激励 | 完成后获得星星，故事显示完成状态 |
| 进度 | 刷新页面后仍能看到完成记录 |
| 时长 | 一个孩子可在 5–10 分钟内完成一篇 |
| 技术 | 本地可运行，可部署为静态网页 |

## 4. 用户主流程

### 4.1 主流程总览

```text
打开首页
→ 查看今日阅读任务
→ 进入故事列表
→ 选择故事
→ 查看故事详情
→ 开始阅读
→ 分页读完故事
→ 进入答题
→ 完成 3 道题
→ 获得星星奖励
→ 返回首页或继续下一篇
```

### 4.2 阅读流程

```text
故事详情页
→ 点击“开始阅读”
→ 进入阅读页第 1 页
→ 阅读当前页内容
→ 查看当前页重点字词
→ 点击“下一页”
→ 读完最后一页
→ 点击“去答题”
```

阅读页规则：

- 每页文本不宜过长。
- 每页最多展示 1–3 个重点字词。
- 当前页需要显示页码进度，例如“2/5”。
- 不强制计时，不给孩子制造压力。
- 可以返回上一页，但不跳过答题流程。

### 4.3 答题流程

```text
进入答题页
→ 展示第 1 题
→ 孩子选择答案
→ 立即显示对错反馈
→ 点击“下一题”
→ 完成第 3 题
→ 进入完成页
```

答题规则：

- 每篇固定 3 道题。
- 第 1 题为识字题。
- 第 2、3 题为阅读理解题。
- 第一版只做单选题和判断题。
- 每题提交后显示简单反馈。
- 不做倒计时，不做复杂扣分。

### 4.4 完成与激励流程

```text
完成全部题目
→ 计算答题结果
→ 发放星星
→ 保存完成状态
→ 展示鼓励语
→ 返回首页或继续阅读
```

奖励规则：

| 条件 | 奖励 |
|---|---|
| 完成阅读和答题 | 1 颗星 |
| 答对 2 题 | 2 颗星 |
| 答对 3 题 | 3 颗星 |

说明：

- 最少完成即可获得 1 颗星，避免孩子受挫。
- 满分获得 3 颗星，形成正向激励。
- 同一篇故事重复完成时，第一版保存最高星星数。

## 5. 页面结构

### 5.1 首页

首页目标：让孩子快速知道今天要做什么。

首页内容：

- 产品名称
- 欢迎语
- 今日阅读任务卡片
- 今日推荐故事
- 总星星数
- 连续阅读天数
- 已完成故事数量
- “开始今日阅读”按钮
- “查看全部故事”入口

首页示例结构：

```text
顶部：儿童互动阅读
欢迎语：今天也来读一个小故事吧！
数据区：星星数 / 连续天数 / 已完成故事
今日任务卡：推荐故事 + 预计 5–10 分钟
主按钮：开始今日阅读
次按钮：全部故事
```

交互规则：

- 点击“开始今日阅读”进入推荐故事详情页。
- 点击“全部故事”进入故事列表页。
- 如果今日推荐故事已完成，推荐下一篇未完成故事。
- 如果全部完成，显示“今天太棒啦，可以复习喜欢的故事”。

### 5.2 故事列表页

故事列表目标：让孩子自己选择故事。

列表展示字段：

- 故事封面或图形占位
- 故事名称
- 等级 L1 / L2 / L3
- 主题标签
- 重点字词
- 预计阅读时间
- 完成状态
- 获得星星数

第一版故事数量：

```text
5 篇原创短故事
```

候选故事：

| 故事名 | 等级 | 主题 | 重点字词 |
|---|---|---|---|
| 小兔子的红雨伞 | L1 | 分享与帮助 | 雨、伞、红、朋友 |
| 会发光的石头 | L1 | 好奇心 | 光、石、找、亮 |
| 小熊不想刷牙 | L1 | 生活习惯 | 牙、刷、水、干净 |
| 云朵邮递员 | L2 | 想象力 | 云、信、送、天空 |
| 森林里的慢慢车 | L2 | 耐心与合作 | 车、慢、等、一起 |

交互规则：

- 点击故事卡片进入故事详情页。
- 已完成故事显示星星和“已完成”。
- 未完成故事显示“开始阅读”。
- 列表默认按等级和推荐顺序展示。

### 5.3 故事详情页

故事详情页目标：让孩子知道这篇故事讲什么、要学什么。

页面内容：

- 故事名称
- 故事主题
- 等级
- 预计阅读时间
- 简短简介
- 重点字词
- 题目数量
- 当前完成状态
- “开始阅读”按钮
- “返回列表”按钮

页面示例：

```text
小兔子的红雨伞
主题：分享与帮助
等级：L1
预计时间：5 分钟
重点字词：雨、伞、红、朋友
今天的问题：3 道
按钮：开始阅读
```

交互规则：

- 点击“开始阅读”进入阅读页第 1 页。
- 如果已完成，可以显示“再读一次”。
- 再读一次不会清空原来的最高星星记录。

### 5.4 阅读页

阅读页目标：让孩子顺利读完故事。

页面内容：

- 故事标题
- 当前页码
- 当前页故事文本
- 当前页重点字词
- 上一页按钮
- 下一页按钮
- 最后一页显示“去答题”按钮

阅读页交互：

- 第一页隐藏或禁用“上一页”。
- 非最后一页点击“下一页”。
- 最后一页点击“去答题”。
- 支持返回故事详情页，但需要提示“阅读还没完成”。

文本原则：

- 每页 1–3 句话。
- 使用短句和清晰标点。
- 避免过长段落。
- 重点字词要和题目有关。

### 5.5 答题页

答题页目标：让孩子完成识字和阅读理解反馈。

页面内容：

- 当前题号，例如“第 1/3 题”
- 题目类型，例如“识字题”或“阅读理解题”
- 题干
- 选项
- 提交按钮
- 对错反馈
- 下一题按钮

题型设计：

| 题型 | 说明 | 示例 |
|---|---|---|
| 识字选择题 | 选择正确的字或词 | “红雨伞”的“伞”是哪一个？ |
| 阅读理解题 | 根据故事内容选择答案 | 小兔子把雨伞借给了谁？ |
| 判断题 | 判断故事信息是否正确 | 小熊最后刷牙了吗？ |

交互规则：

- 未选择答案时不能提交。
- 提交后显示正确/错误。
- 显示一句简单解释。
- 点击“下一题”进入下一题。
- 最后一题完成后进入完成页。

反馈文案示例：

| 结果 | 文案 |
|---|---|
| 正确 | 答对啦！你读得很认真。 |
| 错误 | 没关系，我们再看看故事里的线索。 |
| 完成 | 太棒了，你完成了今天的阅读任务！ |

### 5.6 完成页

完成页目标：给孩子明确成就感，并引导继续阅读。

页面内容：

- 完成标题
- 获得星星数
- 答对题数
- 鼓励语
- 故事完成状态
- 返回首页按钮
- 继续下一篇按钮

完成页示例：

```text
太棒啦！
你完成了《小兔子的红雨伞》
获得：★★★
答对：3/3
继续保持，明天再来读一个新故事吧！
```

交互规则：

- 点击“返回首页”回到首页。
- 点击“继续下一篇”进入下一篇未完成故事详情页。
- 如果所有故事已完成，显示“全部故事都点亮啦”。

## 6. 核心功能模块

### 6.1 故事内容模块

故事内容模块负责保存第一版所有故事和题目。

每篇故事包含：

- 故事 ID
- 故事标题
- 等级
- 主题
- 简介
- 预计阅读时间
- 重点字词
- 分页正文
- 题目列表

第一版内容规则：

- 5 篇原创短故事。
- 每篇 3–5 页。
- 每页 1–3 句话。
- 每篇 3 道题。
- 不使用竞品内容、IP、图片、音频、课程或题库。

### 6.2 阅读模块

阅读模块负责分页阅读体验。

功能包括：

- 显示当前故事标题。
- 显示当前页文本。
- 显示页码进度。
- 显示重点字词。
- 支持上一页、下一页。
- 读到最后一页后进入答题。

### 6.3 识字模块

识字模块负责让孩子在故事语境中认识重点字词。

功能包括：

- 在故事详情页展示重点字词。
- 在阅读页展示当前页重点字词。
- 在答题页设置 1 道识字题。

识字题原则：

- 题目简单明确。
- 不脱离故事语境。
- 不做过难拼音、笔顺和复杂字形分析。
- 第一版以选择题为主。

### 6.4 阅读理解模块

阅读理解模块负责判断孩子是否读懂故事。

功能包括：

- 每篇设置 2 道阅读理解题。
- 题目基于故事内容，不考课外知识。
- 答题后给出简单解释。

阅读理解题方向：

- 谁做了什么。
- 为什么这样做。
- 后来发生了什么。
- 这个故事告诉我们什么。

### 6.5 激励模块

激励模块负责完成反馈和持续动力。

第一版激励元素：

- 星星
- 完成状态
- 连续阅读天数
- 简单徽章

星星规则：

| 答题结果 | 星星 |
|---|---|
| 完成但答对 0–1 题 | 1 颗 |
| 答对 2 题 | 2 颗 |
| 答对 3 题 | 3 颗 |

徽章规则：

| 条件 | 徽章 |
|---|---|
| 完成第 1 篇故事 | 阅读小苗 |
| 累计获得 10 颗星 | 星星收集员 |
| 连续阅读 3 天 | 坚持小达人 |
| 完成全部 5 篇故事 | 故事探险家 |

第一版不做：

- 金币消费
- 商城
- 宠物养成
- 排行榜
- 社交分享

### 6.6 学习进度模块

学习进度模块负责在本地保存孩子的完成记录。

第一版只使用浏览器本地存储。

需要保存：

- 已完成故事 ID
- 每篇故事最高星星数
- 每篇故事最近一次答对题数
- 总星星数
- 连续阅读天数
- 最近一次完成日期

不保存：

- 儿童真实姓名
- 手机号
- 照片
- 语音
- 精确身份信息
- 上传到服务器的数据

## 7. 内容与题目设计

### 7.1 故事内容结构

第一版故事内容建议结构：

```text
开头：出现角色和小问题
发展：角色尝试解决问题
转折：遇到一个小困难
结尾：问题解决，形成温和正向反馈
```

故事主题要求：

- 适合 6–8 岁儿童。
- 情节简单清晰。
- 价值观正向。
- 避免恐怖、暴力、歧视、成人内容。
- 适合做识字和阅读理解题。

### 7.2 识字题结构

识字题数据结构：

```text
题目类型：word
题干：请选择故事里出现过的重点字。
选项：3–4 个
正确答案：1 个
解释：一句简单解释
```

识字题示例：

```text
题干：“红雨伞”的“伞”是哪一个？
选项：伞 / 花 / 云
答案：伞
解释：伞可以在下雨时帮我们挡雨。
```

### 7.3 阅读理解题结构

阅读理解题数据结构：

```text
题目类型：comprehension
题干：根据故事内容提问
选项：3–4 个
正确答案：1 个
解释：一句简单解释
```

阅读理解题示例：

```text
题干：小兔子为什么把红雨伞借给朋友？
选项：因为朋友被雨淋湿了 / 因为它不喜欢雨伞 / 因为它要睡觉
答案：因为朋友被雨淋湿了
解释：故事里朋友没有伞，小兔子想帮助它。
```

### 7.4 每篇故事的数据要求

每篇故事必须包含以下字段：

| 字段 | 说明 |
|---|---|
| id | 故事唯一标识 |
| title | 故事标题 |
| level | L1 / L2 / L3 |
| ageRange | 推荐年龄 |
| theme | 故事主题 |
| summary | 简介 |
| coverEmoji | 第一版可用 emoji 或图形占位 |
| estimatedMinutes | 预计阅读时间 |
| keywords | 重点字词 |
| pages | 分页正文 |
| questions | 题目列表 |

故事数据示例：

```ts
{
```

## 8. 交互与体验原则

### 8.1 儿童使用原则

- 按钮要大，文字要清楚。
- 每个页面只突出一个主要动作。
- 不使用复杂菜单。
- 不让孩子一次看到太多信息。
- 错题反馈要鼓励，不批评。
- 不使用强刺激、强诱导、长时间沉迷设计。

### 8.2 页面交互原则

- 首页主按钮必须明显。
- 故事卡片可点击区域要大。
- 阅读页只保留阅读、翻页和重点字词。
- 答题页提交前必须选择答案。
- 完成页要给明确奖励和下一步。
- 所有页面都要有返回路径。

### 8.3 反馈与激励原则

- 完成比满分更重要。
- 奖励要轻量，不做复杂经济系统。
- 错误反馈要告诉孩子可以再试。
- 星星用于鼓励，不制造排名压力。
- 连续阅读天数用于培养习惯，不做惩罚。

## 9. 数据与状态

### 9.1 需要保存的数据

第一版需要保存到浏览器本地：

| 数据 | 用途 |
|---|---|
| completedStoryIds | 判断哪些故事已完成 |
| storyStars | 保存每篇最高星星数 |
| storyScores | 保存每篇最近答对题数 |
| totalStars | 首页展示总星星数 |
| streakDays | 首页展示连续阅读天数 |
| lastCompletedDate | 计算连续阅读 |

### 9.2 不保存的数据

第一版不保存任何真实儿童身份数据。

不保存：

- 真实姓名
- 手机号
- 地址
- 照片
- 语音
- 学校
- 班级
- 家长联系方式
- 精确设备身份信息

### 9.3 本地进度规则

本地进度规则：

- 使用 `localStorage` 保存学习进度。
- 每次完成故事后更新对应记录。
- 同一故事重复完成，保存最高星星数。
- 总星星数按每篇最高星星数求和。
- 连续阅读天数按完成日期计算。
- 如果清除浏览器数据，进度会消失，第一版接受这个限制。

本地进度结构示例：

```ts
{
  completedStoryIds: ['red-umbrella'],
  storyStars: {
    'red-umbrella': 3
  },
  storyScores: {
    'red-umbrella': 3
  },
  totalStars: 3,
  streakDays: 1,
  lastCompletedDate: '2026-05-03'
}
```

## 产品方案结论

第一版产品要完成的是一个轻量、可运行、可演示的儿童中文互动阅读 MVP。

核心页面：

```text
首页 → 故事列表页 → 故事详情页 → 阅读页 → 答题页 → 完成页
```

核心闭环：

```text
读故事 → 认字 → 理解 → 获得星星 → 保存进度
```

第一版边界：

```text
只做 Web App、本地数据、本地进度、原创内容，不做账号、后端、支付和复杂商业化功能。
```
