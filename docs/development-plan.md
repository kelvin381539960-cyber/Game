## 1. 开发目标

### 1.1 第一版开发目标

第一版开发目标是完成一个可运行、可演示、可部署的儿童中文互动阅读 Web App MVP。

必须跑通主流程：

```text
首页 → 故事列表 → 故事详情 → 阅读 → 答题 → 完成 → 保存进度
```

第一版完成后，应满足：

- 本地可以通过 `npm install` 和 `npm run dev` 启动。
- 浏览器中可以打开首页。
- 可以查看 5 篇原创故事。
- 可以进入故事详情页。
- 可以分页阅读故事。
- 可以完成每篇 3 道题。
- 可以看到答题反馈和星星奖励。
- 可以保存本地学习进度。
- 刷新页面后仍能看到完成状态。

### 1.2 开发边界

第一版只做前端 MVP。

做：

- Vite + React + TypeScript 项目
- 纯前端页面
- 本地故事数据
- 本地进度存储
- 基础响应式样式
- MVP 主流程

不做：

- 后端服务
- 数据库
- 账号登录
- 支付订阅
- 家长端详情页
- 后台 CMS
- 语音识别
- AI 自动生成内容上线
- 复杂动画
- 宠物养成、商城、排行榜

## 7. D5 实现本地进度

### 7.1 目标

使用 `localStorage` 保存学习进度，刷新页面后仍能看到完成记录。

### 7.2 任务清单

创建：

```text
src/utils/progressStorage.ts
```

实现：

- `loadProgress()`
- `saveProgress(progress)`
- `completeStory(progress, result)`
- `resetProgress()`
- `getDefaultProgress()`

存储 key：

```text
child-reading-progress-v1
```

### 7.3 进度规则

- 完成故事后写入 `completedStoryIds`。
- 每篇故事保存最高星星数。
- 每篇故事保存最近一次答对题数。
- 总星星数按每篇最高星星数求和。
- 连续阅读天数根据完成日期更新。
- 徽章根据最新进度计算。

### 7.4 异常处理

- 没有历史数据时返回默认进度。
- localStorage JSON 损坏时返回默认进度。
- 字段缺失时补默认值。
- 浏览器不支持 localStorage 时页面仍能使用。

### 7.5 验收标准

| 验收项 | 标准 |
|---|---|
| 完成保存 | 完成故事后状态保存 |
| 刷新保留 | 刷新页面后完成状态仍存在 |
| 星星保留 | 星星数量不丢失 |
| 最高分规则 | 重复完成只保存最高星星数 |
| 异常可用 | 存储异常不导致页面崩溃 |

## 11. 风险与处理

### 11.1 内容风险

风险：故事或题目不适合 6–8 岁儿童。

处理：

- 故事保持短句。
- 题目只围绕正文。
- 避免恐怖、暴力、歧视和成人内容。
- 全部使用原创内容。

### 11.2 技术风险

风险：一次性组件过多导致状态混乱。

处理：

- `App.tsx` 统一控制主流程。
- 页面组件只处理当前页面交互。
- 通用逻辑放到 `utils/`。
- 第一版不引入复杂全局状态库。

### 11.3 进度保存风险

风险：localStorage 数据损坏导致页面报错。

处理：

- 读取时使用 `try/catch`。
- 数据异常时返回默认进度。
- 字段缺失时补齐默认值。

### 11.4 体验风险

风险：儿童用户不知道下一步点哪里。

处理：

- 每页只突出一个主按钮。
- 按钮文案使用儿童容易理解的语言。
- 完成后明确提示下一步。
