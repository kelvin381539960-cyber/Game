# 儿童互动阅读软件｜任务实施列表

> 仓库：kelvin381539960-cyber/Game  
> 项目目标：创造一个类似“叫叫”方向的儿童互动阅读软件  
> 执行原则：先确认业务方案，再确认产品方案，再确认技术方案，最后进入开发与测试  
> Git 写入规则：任何阶段内容必须先在 ChatGPT 对话中确认，用户确认后才允许写入 Git

---

## 0. 项目执行规则

### 0.1 总原则

本项目不直接进入写代码阶段，必须按以下顺序推进：

1. 业务方案
2. 产品方案
3. 技术方案
4. 开发计划
5. 代码实现
6. 测试验证
7. 发布准备

### 0.2 每个方案的执行方式

每个方案都分两步：

1. 先确认骨架
2. 再补充详细内容

未经用户确认，不允许写入 Git。

### 0.3 Git 写入规则

每一次 Git 写入前，都必须满足：

- 已在对话中展示拟写入内容
- 用户明确确认
- 再执行 Git 写入
