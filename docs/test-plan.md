## 1. 测试目标

本测试方案用于验证儿童互动阅读 Web App MVP 是否达到第一版可运行、可演示、可继续迭代的标准。

核心验证目标：

```text
首页 → 故事列表 → 故事详情 → 阅读 → 答题 → 完成 → 保存进度
```

必须确认：

- 项目可以安装依赖。
- 项目可以本地启动。
- 页面可以正常展示。
- 5 篇故事数据完整。
- 每篇故事可以阅读和答题。
- 答题反馈和星星奖励正常。
- 完成进度可以保存在本地。
- 刷新后仍能看到完成状态。

---

## 2. 测试范围

### 2.1 本轮测试包含

| 模块 | 是否测试 | 测试重点 |
|---|---|---|
| 项目启动 | 是 | `npm install`、`npm run dev` |
| 首页 | 是 | 今日任务、进度、推荐故事 |
| 故事列表 | 是 | 5 篇故事展示、完成状态、星星 |
| 故事详情 | 是 | 简介、重点字词、开始阅读 |
| 阅读页 | 是 | 分页阅读、上一页、下一页、去答题 |
| 答题页 | 是 | 选项、提交、反馈、下一题 |
| 完成页 | 是 | 星星、答对数、徽章、返回首页 |
| 本地进度 | 是 | localStorage 保存和刷新保留 |
| 响应式 | 是 | 手机和桌面基础可用 |

### 2.2 本轮测试不包含

| 模块 | 不测试原因 |
|---|---|
| 后端接口 | 第一版无后端 |
| 数据库 | 第一版无数据库 |
| 账号登录 | 第一版不做账号 |
| 支付订阅 | 第一版不商业化 |
| 真实语音播放 | 第一版未实现 |
| 语音识别评分 | 第一版不做 |
| AI 内容生成 | 第一版不做 |
| 家长端报告 | 第一版不做 |
| 多浏览器深度兼容 | MVP 阶段只做基础验证 |

## 3. 测试环境

### 3.1 推荐本地环境

| 项目 | 建议 |
|---|---|
| Node.js | 18+ 或 20+ |
| 包管理器 | npm |
| 浏览器 | Chrome、Edge、Safari 任一现代浏览器 |
| 设备 | 桌面浏览器 + 手机尺寸模拟 |

### 3.2 启动命令

```bash
npm install
npm run dev
```

### 3.3 构建命令

```bash
npm run build
```

### 3.4 预览命令

```bash
npm run preview
```
